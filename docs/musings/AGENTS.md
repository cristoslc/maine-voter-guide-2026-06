# Musings

Lightweight, stream-of-consciousness notes about the project or its domain. Observations, half-formed ideas, design rationale — anything that doesn't fit in a spec or ADR yet.

## Rules

- Always commit and push musings file changes immediately after writing.
- Musings do not require the same level of structure or review as specs or ADRs.
- Musings can be renamed, split, or deleted as understanding evolves.
