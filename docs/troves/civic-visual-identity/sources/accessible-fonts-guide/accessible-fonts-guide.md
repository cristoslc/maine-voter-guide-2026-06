# Accessible Fonts: ADA & WCAG Font Guide

## What makes a font ADA-compliant?

A font considered ADA-compliant contributes to digital content that is:
- Readable
- Distinguishable
- Usable by people with disabilities

This means clear characters, sufficient size, spacing, contrast, and compatibility with assistive technology — aligning with WCAG 2.1 or 2.2 Level AA.

## Serif vs. sans-serif

- **Serif fonts** — contain decorative elements (marks/lines at stroke ends)
- **Sans-serif fonts** — more block-like, no decorative elements

Sans-serif fonts are generally viewed as more accessible for screen readers, though research is not entirely conclusive. The key is proper implementation regardless of font choice.

## Key rules for accessible typography

### Font size
- 16px is the web standard for body text (approximates 11–12pt in print)
- Scales well across devices and viewing distances
- Meets accessibility requirements

### Character distinction
- Examine lowercase "i", lowercase "l", uppercase "L", and numeral "1"
- Some sans-serif fonts make these nearly identical
- Look for subtle distinctions like serifs on "1" or a tail on "l"

### Line spacing
- Adequate line height improves readability
- Sufficient spacing between paragraphs

### Contrast
- Sufficient contrast between text and background
- WCAG AA requires 4.5:1 for normal text, 3:1 for large text

## Research findings

Proper typography can:
- Improve reading accuracy by up to **20%**
- Reduce eye strain by **30%**

## No official font list

The ADA does not list or approve individual font names. Compliance is judged by how legible and accessible the typography is in context, according to WCAG criteria.

## Civic design context

For voter guides and civic materials:
- Simple sans-serif fonts (Arial, Helvetica, Verdana, Calibri) are recommended by the Center for Civic Design
- Google Noto Sans provides broad language support
- 16px minimum for body text
- Ensure high contrast ratios
- Test with assistive technology
