# Ubiquitous Language

Domain vocabulary organized by bounded context. See `docs/ubiquitous-language/` for full definitions and relationships.

## Research Context

| Term | Definition |
|------|-----------|
| **Source** | A primary document cited in the voter guide (news article, campaign page, transcript, PDF). Registered in `_data/sources.js` with a unique `id`, `url`, `label`, `summary`, `races[]`, and `candidates[]`. |
| **Investigation** | A persistent research thread linking related sources within `rk` |
| **Synthesis** | An agent-generated summary produced by `rk` from aggregated sources |
| **Sidecar** | A Jinja2 template generated by `rk` for an agent to fill in (tags, syntheses, queries) |
| **Evidence Pool** | A collection of research sources backing a specific claim or analysis |

## Election Data Context

| Term | Definition |
|------|-----------|
| **Election** | A specific election event (e.g., "June 2026 Primary"). Root aggregate wrapping all races, ballot questions, and key dates. |
| **Geography** | A physical location in the hierarchy: state → county → municipality → precinct. Each Geography node has an `id`, `name`, `type`, `parent`, and `aliases`. Inheritance walks up the `parent` chain. |
| **Jurisdiction** | A governing entity that runs elections. Maps to a Geography via `geoRef`. Has `geoScope` (state, county, municipal). Races inherit from parent jurisdictions: a South Portland voter sees state-wide, county, and municipal races. |
| **Office** | An elected position that persists across election cycles (e.g., "Governor", "US Senate"). Independent of any specific Race. Has `title`, `officeDesc`, `jurisdiction`, `termLength`, `seatsAvailable`, `type`.|
| **extends** (Office inheritance) | An Office may inherit fields from a base Office via the `extends` key (e.g., `cumberland-county-commissioner-4 extends county-commissioner`). The base provides defaults (officeDesc, termLength, type); the extending office overrides specific fields (title, jurisdiction, districtNote). The `resolveOffice` Eleventy filter walks the chain. |
| **Party** | A political party registry entry: `id`, `tag` (d/r/g/l), `fullName`, `shortName`, `color`. Referenced by Race and Candidate via FK. CSS uses `--party-{id}-*` tokens for party-identified UI elements only. See `docs/user-experience/design-system.md` for the semantic token system. |
| **Race** | This election's contest for an office. References `office` (FK), `party` (FK for partisan races), `election` (FK), `jurisdiction` (FK). Stored once; inherited by child jurisdictions. Contains `positionGroups` with candidate-specific content. |
| **Ballot Question** | A referendum or bond question appearing on the ballot. Linked to a `jurisdiction` and `election`. |
| **Candidate** | A person running for office. Registry in `_data/candidates.js` with cross-race info: `id`, `name`, `party` (FK), `races[]` (FK), `incumbent`, `residence`, `campaignWebsite`, `ballotpediaUrl`. Race-specific content stays in the Race. |
| **Issue** | A controlled vocabulary for position topic labels (e.g., "Taxes", "Healthcare", "Housing"). Enables cross-race comparison. Registry in `_data/issues.js`. |
| **Polling Location** | A physical site where voters cast ballots. Linked to Geography, not Jurisdiction. |
| **Precinct** | A geographic subdivision assigning voters to a polling location. Leaf level of Geography. |

## Voter Guide Content Context

| Term | Definition |
|------|-----------|
| **Voter Guide** | The complete editorial package for one election |
| **Race Summary** | Nonpartisan overview of a contested race |
| **Question Analysis** | Pro/con breakdown of a ballot question |
| **Endorsement** | An organization's stated position on a candidate or question |

## Presentation Context

| Term | Definition |
|------|-----------|
| **Site** | The generated static website |
| **Page** | A single renderable document in the site |

### Avoid

- "Survey" → use **Voter Guide** (surveys collect data; guides inform voters)
- "Issue" (when referring to a ballot item) → use **Ballot Question** (ambiguous; could mean campaign topic or referendum)
- "Issue" (when referring to a topic label) → use **Issue** as the controlled vocabulary term from `_data/issues.js`
- "District" → use **Precinct** for voter assignment; **Geography** for the location hierarchy; **Jurisdiction** for the governing entity
